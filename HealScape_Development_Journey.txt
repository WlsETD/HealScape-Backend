====================================================================
HealScape (癒見) - 區塊鏈與 AI 復健系統專案開發歷程紀錄
====================================================================

【專案概述】
HealScape 是一款結合了 AI 動作辨識、HAPI FHIR 國際醫療標準，以及 Web3 區塊鏈概念 (PoPW - Proof of Physical Work) 的創新復健系統。專案分為三個主要使用者端：病患 (Patient)、治療師 (Therapist) 與管理員 (Admin)。

以下是本專案從零到目前的完整開發與建置流程紀錄：

--------------------------------------------------------------------
第一階段：需求分析與系統架構設計 (Architecture Design)
--------------------------------------------------------------------
1. 確立核心目標：解決居家復健依從性低的問題，透過 AI 監測確保動作正確，並以健康幣 (HEAL) 作為獎勵。
2. 技術選型：
   - 前端：HTML5, CSS3, Vanilla JavaScript, Tailwind CSS (樣式框架), FontAwesome (圖示)。
   - 後端：Node.js, Express.js。
   - 資料庫：基於本地 JSON 檔案的輕量級資料庫 (存放 users, ledger, balances 等)。
   - 外部服務整合：HAPI FHIR (真實醫療數據伺服器)、AI 模型 (TensorFlow.js / MediaPipe)。
3. 定義使用者角色：
   - Patient (病患)：執行復健任務、賺取獎勵、查看趨勢。
   - Therapist (治療師)：監控病患數據、下達處方、查看 FHIR 報告。
   - Admin (管理員)：管理使用者帳號、查看系統日誌。

--------------------------------------------------------------------
第二階段：後端基礎建設 (Backend Foundation)
--------------------------------------------------------------------
1. 專案初始化：使用 `npm init` 建立 `server` 資料夾，安裝必要套件 (express, cors, jsonwebtoken, axios)。
2. 建立伺服器進入點：撰寫 `server/server.js`，設定 CORS、靜態檔案路由與啟動 Port。
3. 設計 JSON 模擬資料庫結構：
   - `server/data/users.json` (帳號密碼與基本資料)
   - `server/data/ledger.json` (區塊鏈帳本紀錄，包含 MINT 與 BURN)
   - `server/data/balances.json` (使用者的健康幣餘額)
   - `server/data/logs.json` (系統操作日誌)
4. 實作核心 API 路由 (`server/routes/`)：
   - `auth.js`：處理登入驗證、Token 發放。
   - `admin.js`：處理新增/修改/刪除使用者，以及獲取系統日誌。
   - `blockchain.js`：模擬 Web3 智能合約，處理健康幣的發放 (Mint) 與銷毀/兌換 (Burn)。

--------------------------------------------------------------------
第三階段：HAPI FHIR 醫療資料伺服器串接 (Medical Data Integration)
--------------------------------------------------------------------
1. 實作 `server/routes/fhir.js` 模組。
2. 透過 `axios` 連線至外部真實伺服器 `https://hapi.fhir.org/baseR4`。
3. 病患同步 (Patient Sync)：在使用者登入時，自動向 HAPI FHIR 搜尋或建立對應的 `Patient` 資源 (Resource)。
4. 復健數據上傳 (Observation Upload)：將 AI 辨識出的肩關節角度 (ROM) 或抓握次數，打包成符合 FHIR 標準的 `Observation` 格式並上傳。
5. 歷史紀錄抓取：提供 API 讓前端能夠抓取特定病患的 FHIR 歷史數據，用於繪製趨勢圖。

--------------------------------------------------------------------
第四階段：前端頁面切版與基礎互動 (Frontend UI/UX)
--------------------------------------------------------------------
1. 建立共用樣式 (`css/common.css`) 與各角色的專屬樣式表。
2. 實作登入頁面 (`index.html`)：包含身分選擇、表單驗證與動畫效果。
3. 實作共用 API 與驗證工具 (`js/api.js`, `js/auth.js`)：封裝 Fetch API 請求，處理 Session 與 Token 的儲存。
4. 建立基礎佈局：
   - 病患端 (`patient.html`)：主視圖、相機畫面的容器、導覽列。
   - 治療師端 (`therapist.html`)：病患列表、篩選器、詳細報告彈窗。
   - 管理員端 (`admin.html`)：數據儀表板、用戶清單、日誌檢視。

--------------------------------------------------------------------
第五階段：核心功能實作 - 病患端 (Patient Portal)
--------------------------------------------------------------------
1. 實作 `js/patient.js`。
2. AI 視覺辨識整合：
   - 導入 `@tensorflow-models/pose-detection` (MoveNet) 進行肩關節角度計算。
   - 導入 `@mediapipe/hands` 進行手指抓握與眼手協調訓練。
   - 在 Canvas 上繪製骨架與互動標靶 (UI Overlay)。
3. 遊戲化與區塊鏈機制：
   - 實作每日簽到、服藥回報、認知訓練 (數學題) 來賺取經驗值 (XP) 與健康幣。
   - 完成 AI 動作後，呼叫後端 API 進行 PoPW (Proof of Physical Work) 驗證，並顯示獲得的健康幣獎勵與假交易哈希 (TxHash)。
   - 實作「兌換折扣」功能 (Burn token)。

--------------------------------------------------------------------
第六階段：核心功能實作 - 治療師端 (Therapist Portal)
--------------------------------------------------------------------
1. 實作 `js/therapist.js`。
2. 實作非同步數據載入：自動從後端與 FHIR Server 抓取所有病患的狀態，並標示風險等級 (Risk Alert)。
3. 繪製趨勢圖表：整合 `Chart.js`，將 FHIR Server 抓取到的 ROM 歷史數據繪製成折線圖。
4. 下達醫療處方：實作彈窗讓治療師能發布智能合約處方，設定每日目標次數與難度。

--------------------------------------------------------------------
第七階段：核心功能實作 - 管理員端 (Admin Portal)
--------------------------------------------------------------------
1. 實作 `js/admin.js`。
2. 實作 CRUD 操作：在前端介面新增、編輯病患或治療師帳號。
3. 系統監控：實作 `setInterval` 每 10 秒自動發送請求 (`GET /api/admin/users`)，實現背景自動刷新數據的功能。
   - *註：當資料未變動時，伺服器會回傳 HTTP 304 Not Modified 以節省頻寬。*

--------------------------------------------------------------------
第八階段：測試、除錯與體驗優化 (Optimization & Bug Fixing)
--------------------------------------------------------------------
1. 狀態管理與視圖切換：優化 SPA (Single Page Application) 的渲染邏輯，使用 `innerHTML` 動態替換畫面。
2. 修復「自動滾動回頂端」問題 (Auto-Scroll Bug Fix)：
   - 問題描述：由於定時器 (如治療師端每 5 秒刷新數據) 或畫面切換會重寫 DOM，導致長列表或長篇報告的滾動條被重置為 0。
   - 解決方案：在 `js/therapist.js` 與 `js/patient.js` 的 `render()` 函式中，為滾動容器 (如 `patient-list-container`, `details-scroll-container`, `blockchain-history-container`) 加入 ID。在重繪前保存 `scrollTop`，重繪後立即還原。
3. 加入測試輔助工具：在病患端加入「快速模擬上傳」按鈕，方便在無攝影機的環境下測試 FHIR 串接與區塊鏈發放流程。
4. 部署準備：建立 `Dockerfile` 與 `docker-compose.yml`，讓專案能夠透過容器化技術快速部署。

--------------------------------------------------------------------
目前狀態 (Current Status)
--------------------------------------------------------------------
- 系統已具備完整的全端架構。
- 成功連線至外部 HAPI FHIR 伺服器進行真實的醫療數據存證。
- 各角色的核心業務邏輯 (復健、監控、管理) 皆已可正常運作且 UI 體驗流暢。
